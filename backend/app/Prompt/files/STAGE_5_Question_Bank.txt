=== QUESTION BLOCK STRUCTURE per evidence item ===

Every evidence item block follows this structure:

[sort_order 0]
  ALWAYS: file upload question for the primary evidence artifact.
  question_type: file
  required: false (never block on file alone)
  accept: derive from Stage 2 evidence_type for this item
  upload_label: 2-4 sentences describing exactly what to upload
  evidence_required_raw: "Document/File Upload"

[sort_order 1]
  Date or metadata question.
  If evidence_type includes diagram/document/policy: date question for document date
  If evidence_type includes spreadsheet/inventory: spreadsheet question for the data table
  If evidence_type includes config/log: text question for tool name and version

[sort_order 2 onwards — DERIVED FROM STAGE 4 CRITERIA]
  For each criterion in Stage 4 sufficiency_criteria.criteria for this item's rows:
    - Convert criterion → select question (Yes/No/Unknown or Confirmed/Partial/Not present)
    - required: true
    - Immediately follow with a textarea for justification ONLY if the compliant
      answer is NOT the expected one. Use show_when_question + show_when_values.

  How to determine which criteria to turn into questions:
    a) Look up all Stage 4 rows where item_code = this item.
    b) Collect the UNION of all criteria across all controls this item serves.
    c) Deduplicate criteria that are essentially the same question.
    d) Convert each unique criterion into one select question.
    e) Add one textarea immediately after any select where a non-compliant answer
       needs explanation (show_when_values = the non-compliant answers).

[second to last sort_order]
  Summary textarea: "Describe any known limitations or scope exclusions for this evidence"
  question_type: textarea, required: false, rows: 4

[last sort_order — ALWAYS]
  question_key: "known_gaps_and_plan"
  label: "Known gaps and remediation plan"
  question_type: textarea, required: false, rows: 4
  guide: "List any known gaps in this evidence and your plan to address them."
