═══════════════════════════════════════════
STATE MACHINE — stateEngine.js
═══════════════════════════════════════════
Maintain this fact store (all default to "unknown"):

const facts = {
  local_swift_footprint: "unknown",       // yes | no | unknown
  owns_comm_interface: "unknown",          // yes | no | unknown
  owns_messaging_interface: "unknown",     // yes | no | unknown
  uses_swift_connector: "unknown",         // yes | no | unknown
  uses_customer_connector_only: "unknown", // yes | no | unknown
  access_only_via_provider_gui: "unknown", // yes | no | unknown
}

Export these pure functions:

1. updateFact(facts, key, value) → newFacts
   - Updates the fact
   - Runs cascading inferences (see DEPENDENCY RULES below)
   - Returns updated facts + array of auto-derived changes with reasons

2. detectContradictions(facts) → contradiction | null
   - Returns { conflictingFacts, message, resolutionOptions } or null

3. classify(facts) → { architecture, confidence, evidence, skipped }
   - Returns best architecture or null if underdetermined
   - Applies escalation: A1 > A2 > A3 > A4 > B

4. nextQuestion(facts, answeredIds) → question | null
   - Returns next applicable question based on current facts
   - Returns null when classification is confident

DEPENDENCY RULES — architectureRules.js
═══════════════════════════════════════════
Implement these as pure rule functions applied after every fact update:

RULE 1 — B short-circuit:
  IF local_swift_footprint = "no" AND access_only_via_provider_gui = "yes"
  AND uses_customer_connector_only = "no"
  → architecture = B
  → mark owns_comm_interface, owns_messaging_interface, uses_swift_connector as N/A

RULE 2 — A1 triggers:
  IF owns_comm_interface = "yes"
  → architecture candidate = A1
  → infer local_swift_footprint = "yes"
  → infer access_only_via_provider_gui = "no"

RULE 3 — A2 candidate:
  IF owns_messaging_interface = "yes" AND owns_comm_interface = "no"
  → architecture candidate = A2

RULE 4 — A3 overrides A4/B:
  IF uses_swift_connector = "yes"
  → architecture candidate = A3
  → infer local_swift_footprint = "yes"
  → infer access_only_via_provider_gui = "no"

RULE 5 — A4:
  IF uses_customer_connector_only = "yes"
  AND uses_swift_connector = "no"
  AND owns_comm_interface = "no"
  AND owns_messaging_interface = "no"
  → architecture candidate = A4

RULE 6 — Escalation (most comprehensive wins):
  Priority: A1 > A2 > A3 > A4 > B
  If multiple candidates, return highest.

CONTRADICTION RULES:
  - access_only_via_provider_gui = "yes" AND uses_swift_connector = "yes"
    → CONFLICT: "No local footprint contradicts running a Swift connector locally."
    → Resolution options: ["I do run it locally (use A3 path)", "I don't run it (revert)"]

  - local_swift_footprint = "no" AND owns_comm_interface = "yes"
    → CONFLICT: "Owning a comm interface implies local footprint."
    → Resolution options: ["Correct, we do have footprint", "We don't own it, I was wrong"]

═══════════════════════════════════════════
COMPONENT BEHAVIOR
═══════════════════════════════════════════

Classifier.jsx (controller):
- Holds facts, answeredIds, contradictions, derivedChanges in useReducer
- On each answer:
  1. Call updateFact() → get newFacts + derivedChanges
  2. Call detectContradictions(newFacts) → show ContradictionAlert if found
  3. Call classify(newFacts) → update candidate architecture display
  4. Call nextQuestion(newFacts, answeredIds) → advance or show ResultCard

QuestionCard.jsx:
- Animated entrance (fade + slide up)
- Shows question text, subtext, option buttons
- "Not sure" always shown as last option if fallback exists
- Disable options while contradiction is unresolved

ProgressTrail.jsx (sidebar or top strip):
- Shows each answered fact as a row: label | answer | [derived badge if auto-set]
- Shows current candidate architecture prominently
- Skipped/N/A questions shown greyed out with reason

ContradictionAlert.jsx:
- Appears as a modal/banner
- Shows: what conflicts, why it matters, resolution options as buttons
- Must be resolved before next question appears

ResultCard.jsx:
- Shows: Architecture Type (large), confidence badge, evidence list
- Shows what questions were skipped and why
- Shows a brief scope summary (pull from architectures.js)
- CTA: "Proceed to Control Assessment" (placeholder button)
